# 일기당천 (一騎當千) — 삼국 서바이버

> 한 명의 장수로 수천의 적병을 베어내는 브라우저 뱀서라이크.
> 뱀파이어 서바이버의 중독 루프 × 삼국무쌍의 쾌감 × Three.js 화려한 이펙트.

- 플랫폼: 브라우저 (데스크톱 + 모바일), Three.js + Vite + TypeScript
- 배포: https://midagedev.github.io/threesur/
- 런 길이: 10분 (사망 시 즉시 결과). 목표: "한 판만 더" 중독 루프.

---

## 0. 방향 리셋 (2026-07-13, 오너 결정 — 최우선 원칙)

> **기능 넓히기를 멈추고 "손맛 + 3D 스펙터클"에 집중한다.** 시스템 개수보다 한 판의 몰입·타격감·화면 스펙터클이 우선. Three.js를 자랑하는 기술 데모급 비주얼 + 닌텐도/대난투식 역동적 움직임·카메라 워크.

- **P0 투사체 가독성**: 블룸에 코어가 타서 "빛나는 광원"만 보이는 상태 금지. 불투명 3D 본체(무기별 뚜렷한 실루엣·색) + 낮은 강도 애디티브 후광 + 방향 트레일. HDR emissive는 후광에만. 적 200+ 밀도 스크린샷으로 검증.
- **오픈필드 회귀**: 성곽 미로 상시 지형은 뱀서 핵심(포위·인파 압박)을 깎음 → 제거. 호로관 파성은 가끔 터지는 세트피스 스펙터클로만.
- **투자 트랙**: ① 이동 손맛(가감속+관성, i-frame 대시+잔상, 스쿼시&스트레치) ② 동적 카메라(위협 밀도/무쌍/보스 반응 줌, 룩어헤드, 펀치줌) ③ 3D 스펙터클(킬 포인트라이트, 리본 트레일, 지면 충격파 리플/데칼).
- **덜기**: 무한모드 프롬프트는 결과 화면으로 이동, 동시 활성 시스템(저주+동료+이벤트+오브젝트) 등장 빈도·밀도 축소.

## 1. 코어 루프

1. 장수 선택 → 10분 생존전 시작
2. 이동만 조작 (WASD/조이스틱). 무기는 전부 자동 발동
3. 적 처치 → 혼백 젬 드랍 → 레벨업 → 3택 업그레이드 (무기/패시브)
4. 1.5분마다 엘리트(A급 장수), 3/6분 중간보스, 9분 최종보스 여포
5. 선택 장수는 런 시작·보스·무쌍 때 군웅전 NPC 카드 대사를 말하고, 45초에는 장수별 원군 1명이 합류 → 추종 자동 공격 + 합류 대사
6. 킬 콤보 → 무쌍 게이지 → Space로 무쌍난무 (화면 청소 궁극기)
7. 사망/클리어 → 골드 정산 → 메타 상점(영구 강화) → 재도전

## 6. 성장/경제

- 혼백 젬: 파랑(1) · 초록(5) · 빨강(20), 픽업 반경 기본 2m, 젬은 자석 흡인 연출
- 레벨: `nextXp = 5 + level×10`, 레벨업 시 일시정지 + 3택 카드 (+골드 리롤 1회 50G)
- 킬 콤보: 3초 내 킬 유지, 100/500/1000 돌파 시 "百人斬!" 배너 + XP 보너스, 콤보가 무쌍 게이지 가속
- 무쌍 게이지: 킬 +1%, 피격 +3%. 발동 시 5초: 적 시간 0.3×, 무적, 장수별 난무, 종료 충격파
- 골드 → 메타 상점 (localStorage): 공격/체력/이속/반경/쿨다운/시작레벨/부활 각 5단계, 비용 `200×2^n`. 여포 해금 5000G
- 도감: 처치한 보스/엘리트 기록, 최고 기록(생존시간·킬수·레벨) 표시

## 7. 비주얼 디렉션 — "Fable 데모급"

- **무대**: 밤의 전장. 어두운 남흑색 지면(절차 생성 수묵 노이즈 텍스처 + 은은한 격자), `FogExp2`, 화면 곳곳 떠다니는 불씨·반딧불 파티클
- **맵**: 30m 청크 방/안뜰을 폭 7m 성문과 도로로 연결. 실제 성벽 충돌 + 적 flow-field 길찾기 + 막힌 스폰 투영
- **호로관**: 1024×512 대형 랜드마크. 문 앞 12킬 시 실제 길이 열리고 3D 픽셀 파편·충격파·후속 돌격대가 이어지는 파성 세트피스
- **카메라**: PerspectiveCamera FOV 40, 상방 55° 내려보기, 플레이어 부드러운 추적, 트라우마 기반 셰이크 + 킬 펀치줌
- **렌더링**: ACESFilmicToneMapping, UnrealBloomPass(strength≈0.9, threshold≈0.6) — 모든 이펙트는 HDR emissive로 블룸을 태운다
- **스프라이트**: NearestFilter, 세워진 빌보드 쿼드(살짝 기울임), 발밑 타원 블롭 그림자, 걷기 4프레임 8fps
- **적 렌더**: 시트별 InstancedMesh + 커스텀 셰이더 (인스턴스 속성: uv 프레임, 피격 플래시, 스케일, 틴트) — 600+ 마리 60fps
- **타격감**: 피격 화이트 플래시, 넉백, 엘리트/보스 히트스탑 40ms, 사망 파티클 버스트 + 디졸브
- **공격 아트**: 생성 원화를 제한 팔레트와 큰 픽셀 클러스터로 정리한 레트로 스프라이트 18종. 플레이어 기본 연노 화살과 용담창·언월도·장팔사모·방천화극 공격에 더해 적 화살비·책사 부적·보스 화염탄·독 구체·번개 창·중탄을 전용 실루엣으로 분리했다. 기본 공격은 22m 안의 최근접 적을 자동 조준하고, 스프라이트 본체 위에 Three.js 후광·속도선만 합성한다.
- **무기 이펙트**: 불투명 도트 스프라이트를 본체로 두고, 애디티브 후광은 낮은 강도로 뒤에 배치한다. 종류별 잔광 파티클·명중 파편·참격/팔괘/기마 충격파 링을 짧게 겹쳐 모양과 타격 위치를 가리지 않으면서 화려함을 더한다.
- **무쌍 연출**: 시간 슬로우, 비네트+채도 시프트, 방사형 스피드라인, 대형 한자 "無雙" 플래시, 종료 대형 충격파
- **데미지 숫자**: 캔버스 스프라이트 풀, 크리티컬은 크고 노랗게
- **UI 톤**: 먹빛 배경 + 금색 헤어라인, 한자 병기 (예: "무쌍 無雙"), 세리프 명조 계열

## 9. 기술 아키텍처

```
src/
  main.ts              # 부트스트랩, 씬 상태머신 (Title→Select→Run→Result)
  core/   loop.ts input.ts rng.ts events.ts save.ts audio.ts
  gfx/    renderer.ts(postfx) camera.ts atlas.ts sprites.ts(instanced)
          particles.ts(GPU) effects.ts(slash/lightning/ring/trail) ground.ts damageText.ts
  game/   run.ts player.ts enemies.ts(pool+grid) spawner.ts collision.ts
          weapons/*.ts pickups.ts levelup.ts musou.ts boss.ts combo.ts stats.ts
  data/   heroes.ts weapons.ts passives.ts waves.ts upgrades.ts spriteManifest.ts
  ui/     hud.ts screens.ts(title/select/result/shop) cards.ts joystick.ts
public/assets/sprites/      # sgrade/apriority/soldiers/variants 시트 + manifest.json
public/assets/projectiles/  # 생성 원화를 레트로 팔레트로 가공한 투사체·기본 공격 PNG 18종
public/assets/world/        # 512×512 월드킷 + 1024×512 호로관 랜드마크
```

- 적/젬/파티클: 고정 크기 풀 + freelist, 프레임당 할당 0 목표
- 충돌: 균일 그리드(셀 2m) 스페이셜 해시 — 적↔적 분리(soft), 적↔플레이어, 무기 판정은 원/부채꼴/캡슐 수학 판정
- 게임 시간과 렌더 시간 분리(무쌍 슬로우는 적 dt에만 적용), 고정 dt 60Hz 시뮬 + 보간 없이 단순 가변 dt(클램프 33ms)도 허용
- 저장: localStorage key `threesur-save-v1` (골드, 메타 레벨, 해금, 도감, 최고기록)
- 성능 목표: 데스크톱 60fps @ 적 600, 모바일 30fps+ (DPR 클램프 2, 모바일 블룸 해상도 절반)

## 10. 재미 극대화 요소 (Phase 4)

### 전장 이벤트 (런에 변주와 서사를 주는 랜덤 사건, 화면 상단 경고 배너 + 전용 사운드)
- **보급 마차** (2~8분 사이 랜덤 2회): 발광하는 마차(runner 스프라이트 + 수레 연출 + 금색 트레일)가 화면을 가로질러 도주. 처치 시 골드 대량 + 보물상자. 놓치면 사라짐 — 추격의 긴장감.
- **황건 대군 러시** (4분·7분): 30초간 한 방향에서 대군 돌진(일직선 웨이브). 처치 밀도가 높아 콤보/젬 파밍 기회.
- **유성우** (6분 이후 랜덤): 15초간 하늘에서 화염 운석 낙하(경고 데칼 → 착탄 폭발). 적도 플레이어도 피해 — 지형 리스크.

### 전장 오브젝트 (맵에 산재, 상호작용)
- **화약통**: 무기로 터뜨리면 대폭발(반경 4m, 연쇄 유폭). 스폰율 낮게, 위기 탈출 수단.
- **만두 수레**: 파괴 시 만두 드랍(HP 30% 회복). 5분 이후 희소.
- **군신 사당**: 접촉 시 30초 버프 1종(공격력 +30% / 이속 +25% / 무쌍 게이지 2배 충전). 무너진 석상 연출, 사용 후 재사용 불가.
- **파괴 목책**: 무기로 부수면 작은 충격파와 함께 전선을 정리.
- **실제 성벽/성문**: 플레이어와 적 모두 충돌. 문과 갈림길을 따라 다른 안뜰로 이동하며 호로관 파성 후 새 통로가 열린다.

상세한 배치·전투 패턴·페이싱·성능 기준은 [2.5D 게임 구현 가이드](./2_5D_GAMEPLAY_GUIDE.md)를 따른다.

## 15. 회복 이코노미 재설계 + 맵 회복 기믹 (오너 지시 2026-07-14)

> 오너: "회복 수단이 참 적다. 피가 안 닳은 상황에서는 괜찮은데 닳기 시작하면 훅 죽는다. 맵에 체력 회복 기믹도 보고 싶고, 맵 오브젝트에 효과가 없는 것도 있는 듯."

> 랜드마크 군영/망루/잔해는 완전 무기능(이름표만) — 지적 정확.

### 15.1 만두 수레 조기화 + 저체력 연민 보정
- 등장 시점 300s → **90s**. 기본 확률 유지하되, 플레이어 HP<40%면 스폰 롤에서 만두 가중치 상향(연민 보정) — 잘할 땐 변화 없고 위기에만 손을 내밈.

### 15.2 적 드랍 회복 픽업 (뱀서 표준 문법)
- 적 처치 시 낮은 확률(~1.2%)로 소형 회복 픽업(만두 饅頭 소형, 최대체력 8~10%) 드랍. 젬과 동일 자석/픽업 경로 재사용.
- 저체력(HP<40%) 시 드랍률 ×2. 엘리트/장수급은 확률 상향. 상한: 화면 동시 6개.
- 시각: 젬(청/금)과 혼동 없는 온백색 김 오르는 점 — 소형, 절제.

### 15.3 군영 軍營 = 야전 취사장 (맵 회복 기믹 — 무기능 오브젝트에 역할 부여)
- 군영 반경 ~5m 안에 있으면 **초당 최대체력 2.5% 회복** + 취사 연기 파티클/온색 광원 오라.
- 리스크/리워드: 적은 계속 몰려옴 — "버티며 회복할 것인가"의 전술 선택지. 전투 이탈 후 귀환 동선도 생김.
- 쿨타임 없음(위치 자체가 코스트). Three.js 포인트: 연기 기둥 + 지면 온색 라이트필드 오라(회복 중임이 화면에서 읽히게).

### 15.4 잔여 무기능 오브젝트 (경량 역할 — 절제)
- 전고 戰鼓: 접근 시 북 울림 1회(쿨 25s) — 주변 적 0.8s 스턴 + 저음 SFX. (회복과 별개의 유틸 포인트)
- 망루 望樓/공성 잔해: 차폐물 역할 유지(콜라이더 확인), 추가 기믹 없음 — 전부 기능화하면 산만.

### 15.5 밸런스 게이트
- 목표: "닳기 시작해도 플레이로 만회 가능" — 카이팅 봇 10분 생존률 상승, 단 무피격 플레이가 최적해로 남게 회복 총량은 초당 피해 대비 열세 유지.
- 검증: 15.1~15.3 적용 후 카이팅 봇 생존 커브 재측정(3분벽 완화 확인), 회복 픽업 화면 과밀 없음.

## 18. 적탄(잡몹 원거리) 고도화 — 예고·시인성·피격감 (오너 지시 2026-07-14)

> 오너: "잡몹의 원거리 공격이 거의 피하기도 힘들고 잘 보이지도 않고 피격감도 없다."
> 3중 문제를 각각의 레버로: 회피 가능성=발사 예고, 시인성=탄 자체 강조, 피격감=피격 순간 방향성 피드백.

### 18.1 발사 예고 (회피 가능성)
- 사수(원거리 잡몹)가 발사 **0.5s 전 차지 텔레그래프**: 사수 스프라이트 마젠타 플래시 점증 + 사수→플레이어 방향 희미한 조준선(0.3 알파, 지면). 이 0.5s가 회피 판단 창.
- 탄속 소폭 하향(~12%) — 예고를 보고 움직이면 피해지는 수준. 밀도는 불변(스폰 로직 무변경).

### 18.2 탄 시인성 (P0 가독성 게이트 강화)
- 탄 본체: 마젠타 코어 + 진홍 외곽 글로우 강화, 크기 +20%. 야간 먹빛 지면 위에서 1순위로 읽히게.
- 지면 그림자 점(기존): 탄 고도와 무관하게 위치 추적 강화 — 탄과 그림자 세트로 입체 궤적이 읽히게(Three.js 포인트).
- 검증: 밀집 씬에서 적탄이 아군 투사체·젬과 3중 구분(스크린샷), 색약 안전 확인(형태 차이 병행 — 적탄만 원형+꼬리).

### 18.3 피격감 (맞았다는 체감)
- 적탄 피격 순간: (a) 피격 방향 스파크 버스트(탄 색) + (b) 플레이어 넉백 넛지 0.8(방향성 — 어디서 맞았는지 몸으로 알게) + (c) 소형 트라우마 0.15 + 전용 피격 SFX(둔탁, 기존 playerHit과 구분).
- 기존 비네트(피해 비례, #29)와 합산 — 방향성 정보가 핵심 추가분.

### 18.4 게이트
- 회피 검증: 봇이 예고 후 수직 이동 시 회피율 유의미 상승(전/후 피격률 비교).
- 절제: 조준선·차지 플래시가 밀집 씬 시각 소음이 되지 않게(알파 상한, 동시 표시 수 상한 8).

## 19. 닌텐도 감각 팩 (오너 승인 2026-07-14)

> 오너: "마리오카트스러운 무빙, 대난투, 닌텐도스러움 보강." 4종 승인 — 전부 순간 연출·기존 시스템 확장(절제 원칙 내).

### 19.1 드리프트 미니터보 (마리오카트 무빙)
- 대시 중 조향(입력 방향이 대시 방향과 40°+ 벌어짐)하면 **드리프트 상태**: 지면 스키드 마크 데칼 + 말발굽 불꽃 파티클(청→금 2단계 차지).
- 드리프트 0.5s+ 유지 후 해제 시 **미니 부스트 0.4s +30%** (2단계 차지 시 0.6s +40%).
- 구현: player.ts 대시 확장 + effects 스키드/불꽃. 쿨다운 없음(대시 쿨다운이 게이트). 조작 미숙련자에겐 그냥 대시(기존과 동일) — 숙련 레이어.

### 19.2 KO 홈런 별 (대난투 마무리)
- 강타·무쌍 킬 또는 넉백 8+ 킬 시: 적 스프라이트가 **포물선으로 화면 밖까지 날아가며 회전·축소** + 반짝 별 트윙클 + 휘파람 낙하 SFX.

- 구현: enemies 사망 경로 분기 — 시체 페이드 대신 발사체 모드(수직 속도+중력+회전, SoA 재사용).

### 19.3 젬 콤보 피치 상승 (마리오 코인)
- 젬 연속 픽업(간격 <0.8s) 시 픽업음 반음계 상승(최대 +12), 끊기면 리셋. 콤보 피버 중엔 베이스 피치 +2.
- 구현: audio.ts 픽업 SFX에 피치 파라미터 + run 픽업 카운터.

### 19.4 피격 스쿼시&스트레치
- 적 피격 순간 인스턴스 스케일 x1.25/y0.75 → 0.12s 스프링 복원. 보스는 x1.1/y0.92(무게감).
- 구현: sprites 인스턴스 스케일 속성(이미 존재 시 재사용) + enemies 피격 훅. 프레임당 비용 0.

### 19.5 게이트
- 드리프트: 조작감 회귀 없음(비드리프트 대시 동일), 스키드 데칼 수명 짧게(≤2s)·동시 상한.
- KO 별·스쿼시: 밀집 씬 시각 소음 검증(hotFrac·가독성), 60fps 유지.
- 피치 상승: 연속 픽업 폭주 시 귀 아프지 않게 상한 +12 고정.

### 19.6 근접 무기 손맛 보정 (오너 추가 지시 — "근거리가 상대적으로 아쉬움, 팩과 섞어 보완")
- 닌텐도 팩의 타격 유희를 **근접 무기(참격 아크 계열: 용담창/청룡언월도/장팔사모/방천화극/참마검 등)에 우선 배분**:

  2. **스쿼시 강화**: 근접 히트 스쿼시 x1.35/y0.68(원거리 x1.2) + 근접 히트만 마이크로 히트스톱 30ms — "베었다"는 질량감.
  3. **드리프트 차지 임팩트**: 미니 부스트 중 근접 히트 = 피해 +25%·넉백 +50%·KO 별 확정 — 마리오카트 부스트→대난투 스매시로 이어지는 이동-타격 루프. (부스트 0.4~0.6s 한정이라 DPS 영향 미미, 조작 보상)
- 수치 밸런스는 3번의 순간 보정 외 무변경 — 체감 보완이 목적. 봇 킬셰어 재측정으로 근접/원거리 격차 관찰만.
- **(오너 보강) 대시-넉백 2단**: ③의 차지 임팩트와 별개로, **일반 대시 중 근접 히트도 넉백 +50%** — 대시로 파고들어 밀쳐내는 기본 손맛. 드리프트 부스트 중엔 ③(피해+25%·넉백+50%→합산 아님, 상위 대체·KO 별 확정)이 적용되는 2단 구조: 대시=밀침, 부스트=스매시.

### 14.5 보스 그로기 윈도우 — 근접 보스전 성립 (오너 지시 2026-07-14)
> 오너: "보스한테 근거리 딜 넣는 게 거의 불가능." 원인: 넉백 90% 저항 + 돌진/장판 패턴 = 근접 안전 창 부재.
- **패턴 종료 후 그로기 1.8s**: 보스가 시그니처 패턴(dash/rush 돌진 끝, fan/delaybolt 일제사격 직후, firezone 시전 후)을 마치면 **숨 고르기 상태** — 이동 정지, 스프라이트 슬럼프 틴트(회청), 머리 위 "허점 隙" 표시.
- 그로기 중: **받는 피해 +30%, 근접 히트는 +60%** — 회피→접근→베기 리듬이 성립. (몬헌/젤다 문법)
- 발동 주기: 패턴 사이클당 1회(연속 그로기 없음), 보스 HP 25% 이하에선 그로기 1.2s로 단축(막판 긴장 유지).
- 시각(쇼케이스): 그로기 진입 순간 보스 스쿼시(19.4 강화판) + 먼지 훅 + 隙 한자 부유 표시. 절제 — 광원 펄스 없음.
- TTK 게이트는 #40과 합산 측정: 오토에임 가중치 + 그로기 보너스 포함 30~60s. 근접 로드아웃 별도 측정 추가(근접 TTK가 원거리의 1.3배 이내).

### 14.6 보스 패턴 지면 텔레그래프 — 전 패턴 피격 범위 사전 표시 (오너 지시 2026-07-14)
> 오너: "일반적인 시스템처럼 피격 범위가 사전에 나오면 근거리하다 피하거나 하면 좋을 텐데 그런 게 없다."
> 사마의 delaybolt의 예고 데칼 문법을 **전 보스 시그니처 패턴으로 확장** — 근접이 붙어 있어도 위험 지역을 보고 반 발짝 옆으로 피하는 플레이 성립.
- **dash/rush(하후돈·손책·전위 등)**: 돌진 시작 0.7s 전 경로 직사각형 텔레그래프(진홍 반투명 채움 + 테두리, 돌진 방향으로 스윕 채워짐) → 채움 완료 순간 돌진.
- **fan(원소)**: 부채꼴 콘 텔레그래프 0.6s — 발사 각도 범위 표시.
- **firezone(동탁·주유)**: 장판 생성 위치 원형 텔레그래프 0.8s(장판 활성 전) — 기존 delaybolt 원형과 동일 문법.
- **lvbu(여포)**: 조종 돌진 특성상 경로 예고 대신 전방 짧은 콘 0.5s.
- 시각 규약: 적탄 지면 링(점)과 구분되는 **면(area fill) + 진홍 테두리 펄스**, 알파 0.22~0.3, 채움 진행이 남은 시간을 알려줌(추가 UI 없이). 절제: 텔레그래프는 광원 없음, 동시 상한 자연 제한(보스 1패턴씩).
- 근접 루프 완성: 텔레그래프 보고 회피(14.6) → 패턴 종료 그로기(14.5) → 접근·베기 — 회피/펀ish 리듬.
- 검증: 각 패턴 텔레그래프→히트 판정 일치(표시 범위 밖에서 피격 없음), 근접 봇 회피율 전/후.

## 20. 낙양 공방전 — 공성→점령→수성 풀 아크 (오너 승인 2026-07-15)
> 정적 성곽(낙양 외성, DESIGN 12/battlefieldMap)을 **완전 선택형 사이드 오브젝트**로 활성화한다. 성에 접근하지 않는 런에는 어떤 영향도 없다(수비대·판정·수성은 성 근접 게이트 안에서만 구동, 오픈필드 스폰/보스/이벤트 무변경). 소유권 전환·시각화는 markers gfx가 `castleRenderData`로 자기완결 소비.
- 구현: `src/game/siege.ts`(SiegeSystem 상태머신), `battlefieldMap.ts`(성문 봉쇄 콜라이더 + 성문별 파성 카운트 + 내성 판정), `spawner.ts`(수비대/탈환군 스폰 + 수성 스폰율 하향), `objects.ts`(거점화 보급 위치 배치), `boss.ts`(성주 화웅 정의), `src/data/siegeData.ts`(대사·배너 ko/en). run.ts/main.ts는 배선만(콜백 주입).

### 20.1 상태머신 / 타이밍
- 상태: `ENEMY_HELD → BREACHED → LORD → CAPTURED → COUNTERATTACK → HELD(사수 성공, 종결) | FALLEN(함락, 종결)`. **1회성**(재탈환 없음).
- `ENEMY_HELD`: 런 시작 상태. 외성 3성문(castle-s/e/w) + 내성문(castle-keep-s) 봉쇄(콜라이더). map.reset이 재봉쇄 + 소유권 적색 복원(flipVersion++).
- `ENEMY_HELD→BREACHED`: 외성문 ≥1 파성(run.onGateBreached가 castle-* 키로 siege.notifyGateBreach).
- `BREACHED→LORD`: 외성문 ≥1 파성 + 플레이어 외성 내부 진입(내성 밖) + **보스 비활성**. → 내성문 봉쇄 해제 + 성주(화웅) 출진(안뜰 내성문 앞). `onLordSpawn(x,z)`.
- `LORD→CAPTURED`: 성주 처치(run.handleKill 성주 분기 → siege.captureNow). 일반 보스 보상 경로 우회, 보상은 onCapture에서.
- `CAPTURED→COUNTERATTACK`: 점령 +100s. **연기 조건**: (a) 보스전 중, (b) 호로관 활성 중, (c) gameTime이 180/360/540의 25s 전 이내([t-25,t)) — 해소될 때까지 대기.
- `COUNTERATTACK→HELD`: 75s 경과 && 함락 게이지 < 12. `onDefended()`.
- `COUNTERATTACK→FALLEN`: 게이지 ≥ 12 || 플레이어 성 중심 55m 밖 이탈. `onLost()`.
- 미니보스 인프라 재사용: 성주는 Boss(패턴 dash + 텔레그래프 + 그로기), HP=미니보스급(5200)×`(1+0.08·minute)`, minute 최소 1 바닥.

### 20.2 공성 / 점령 수치
- **파성**: 봉쇄된 성문 9m 내 킬을 **성문별로** 카운트, 12킬 도달 시 해당 성문 파성(호로관 recordKillAt 일반화, 한 킬은 한 성문에만 기여). 반환 GateBarrier.key(`castle-s/e/w`)로 run이 호로관/낙양 분기.
- **수비대(garrison)**: ENEMY_HELD/BREACHED 동안 **플레이어가 성 중심 40m 이내일 때만** 안뜰(외성 내부·내성 밖) 랜덤 지점에 트리클 스폰(1.2/s, 성내 생존 로컬 캡 40). pickType(minute) 재사용 + 동탁군 색군(faction 1) 고정.
- **파수꾼**: 성 최초 접근 시 안뜰 4모서리에 궁수(general_bow, 동탁군) 4기 1회.
- **성주(화웅 華雄)**: 동탁의 선봉장. 스프라이트는 동탁군 계열(sgrade #4) 진홍 틴트 재사용(전용 스프라이트 sprite 레인 후속 후보). 처치 시 `castleRenderData.allied=true; flipX/Z=성주 사망 지점; flipVersion++` + `onCapture(x,z)`(run: 배너 '낙양 입성 洛陽入城' + 명기 3택 + 소량 젬 버스트).
- **거점화(CAPTURED~)**: 내성(중심) 회복 오라 반경 6m·초당 2.5%(군영 오라 동일률, run의 kind===5 루프에 얹음). 만두 2개 즉시 + 90s 리스폰(1개, 과다 회복 방지) + 동라 1기.

### 20.3 수성 수치 / 함락 조건
- **개전**: `onCounterattack()`(배너 '낙양 탈환군 來襲' + gfx). 지속 75s. spawner.setSiegeActive(true) → 오픈필드 일반 스폰율 **0.55x**(setBossActive 패턴).
- **웨이브**: t=0/25/50에 3성문 밖 5m 지점 각각 돌격군 10기(2기 궁수, 동탁군). **파성된 성문으로만 진입 가능**(막힌 문 밖에 쌓이면 flow가 파성문으로 우회).
- **트리클**: 1.2/s, 파성된 성문 방향에서 1기(15% 궁수).
- **불화살 일제사**: 8~11s 간격, 플레이어가 성내일 때 산탄 5~7발, 착탄 예고 0.9s. `onVolley(shots{x,z,t=0.9})`. **판정은 siege 소유**: t 경과 시 플레이어가 착탄점 1.7m 내면 피해 10(run.onPlayerHit 경로). 시각화는 gfx 레인(동일 payload).
- **함락 게이지**: 적이 내성 사각 진입 시 개체당 1회 +1(0.25s 스로틀 스캔), 12 도달 → FALLEN. 플레이어 성 중심 55m 밖 이탈도 FALLEN.
- **사수 성공(HELD)**: `onDefended()`(배너 '낙양 사수 洛陽死守' + 대량 젬 + 만두 3 + 무쌍 +50% — run 배선). 거점화 오라 유지(종결 후에도 아군 요새).
- **함락(FALLEN)**: `castleRenderData.allied=false; flipVersion++`(원점=내성 중심) + `onLost()`(배너 '낙양 함락 洛陽陷落', 거점화 해제).

### 20.4 밸런스 가드
- **완전 선택형**: 성 미접근 런은 무영향 — 수비대는 40m 게이트, 판정/오라/보급은 성내 좌표에만 국한, 오픈필드 스폰율은 수성 개전(setSiegeActive) 전까지 무변경.
- **스폰 분리**: 성 콘텐츠 스폰은 전용 경로(spawnSiegeAttacker/Rush/GateWatch, 동탁군 고정). 성주는 단일 Boss 슬롯 점유 → 타임라인 보스와 동시 등장 불가(출진은 `!bossActive` 게이트, 개전은 보스/호로관/보스직전 25s 연기).
- **스케줄 회피**: 수성 개전은 보스전/호로관/보스 슬롯 25s 전을 피해 화면 과밀·이벤트 충돌 방지. 점령 후 언제 개전할지는 상황이 비면 즉시.
- **소프트 캡 준수**: 수성 웨이브/트리클 총량(웨이브 90 + 트리클 ~90)은 ENEMY_SOFT_CAP 420 아래로 자연 수렴, 성주 HP는 미니보스급으로 근접 그로기+#40 오토에임과 합산해 TTK 게이트 내.

### 20.5 Three.js 쇼케이스 포인트

**깃발 점령 웨이브 — 버텍스 셰이더 활용의 정석.** 성 소유가 바뀌면 붉은 군기들이 점령 원점에서부터 거리 비례 딜레이로 차례차례 청록으로 물든다. 색과 펄럭임 진폭은 CPU가 아니라 per-instance 어트리뷰트(`aColor`·`aFlutter`)로 버텍스 셰이더에 흘려보내, 24개 깃발이 단 한 번의 인스턴스 드로우콜로 각자 다른 타이밍에 전환된다. 전환 순간 펄럭임 진폭을 잠깐 1.15배로 키워(sin 엔벌로프) 바람이 웨이브를 밀고 지나가는 인상을 준다 — 파티클 한 톨 없이 정점 데이터만으로 만드는 연출이라 비용이 사실상 0이다. 색은 블룸 임계(0.6) 아래(최대 0.46)로 눌러 천이 광원처럼 타지 않게 했다. 전환 대상은 성곽(외성 경계 +2.5m) 안의 붉은 군기만 — 오픈필드 군영기·내성 금기(황실 상징)는 유지된다.

**불화살 낙하 — 인스턴싱 + SDF 텔레그래프 재사용.** 일제사 화살은 InstancedMesh 한 장(상한 28)에 SoA 풀로 담아, 하늘에서 정확히 착탄 시각 t에 목표점으로 떨어지며 게임 피해 판정과 프레임 단위로 동기된다. 착탄 예고 원은 신규 셰이더를 짜는 대신 보스 패턴이 쓰던 SDF 텔레그래프(단일 셰이더의 shape/progress 유니폼)를 그대로 재사용해 시스템 응집을 높였다. 살대는 어둡게, 촉만 블룸에 살짝 걸리는 온색으로 눌러 다수 낙하에도 화이트아웃이 나지 않는다(실측 hotFrac 0.0008).
